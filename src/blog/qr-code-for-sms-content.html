An SMS QR code (sometimes called \u201csms:\u201d QR) opens a pre-filled text message when scanned. The phone number and message body are encoded in the code. The user scans, their messaging app opens with everything pre-populated, and they hit send. It\u2019s the frictionless way to collect SMS opt-ins, run text-based promotions, and let customers reach you without typing a number.

This guide explains how SMS QR codes work, when to use them, and the format requirements. It takes about 5 minutes to set up.

What Is an SMS QR Code?

An SMS QR code is a standard QR code that, when scanned, opens the user\u2019s default messaging app with a pre-filled text message. The phone number (recipient) and message body are encoded in the code. The user reviews, edits if needed, and hits send.

The format follows the standard SMS URI scheme:

sms:+15551234567?body=Hello%20I%20want%20to%20learn%20more

When a phone scans a QR code containing this URL, the messaging app opens with the phone number and body pre-populated.

When to Use an SMS QR Code

SMS QR codes are useful in any context where you want to make it easy for users to text you:

  • Text-to-join promotions. \u201cText JOIN to 12345 to receive offers\u201d \u2014 a QR code version makes it one tap instead of typing a number and keyword.
  • Customer support. \u201cText us at 555-1234\u201d on a business card or receipt. Users scan and text without dialing.
  • Event check-in. \u201cText CHECKIN to 12345 at the door\u201d \u2014 no app to download, just text the keyword.
  • Lead capture forms. \u201cText INFO to get a free quote\u201d on a flyer.
  • Restaurant waitlists. \u201cText WAITLIST to join the queue\u201d at the host stand.

SMS QR codes are especially useful for audiences that don\u2019t use apps \u2014 older users, customers without smartphones that can install apps, or markets where SMS is the dominant communication channel.

The SMS Format Explained

The SMS URI supports 2 fields:

sms:PHONE?body=BODY

Phone number (required)

The recipient phone number. Use the full international format starting with + and the country code:

sms:+15551234567

The + and country code make the code work in any country. Without the country code, the message goes to a local number that may or may not be the intended recipient.

Body

Pre-fills the message body. Keep it short \u2014 most messaging apps show a character counter, and users don\u2019t want to receive a 200-character pre-filled message:

sms:+15551234567?body=JOIN

Note: spaces and special characters must be URL-encoded. %20 is a space, %0A is a newline.

How to Build an SMS QR Code

Most QR code generators support an \u201cSMS\u201d or \u201cText\u201d format option. You enter the phone number and message body, and the generator creates the QR code.

If you want to build it manually:

  1. Start with sms:+COUNTRYCODEPHONENUMBER
  2. Add ?body=YOUR%20MESSAGE (URL-encode any spaces)
  3. Paste the complete URL into a QR code generator
  4. Set the format to \u201cURL\u201d
  5. Test on multiple phones

Test specifically that:

  • The messaging app opens (not the browser)
  • The phone number is correct
  • The body is pre-filled

International Phone Numbers

SMS QR codes work internationally if you use the full international format. A US number is +1 followed by the 10-digit number:

sms:+15551234567

A UK number is +44 followed by the number without the leading 0:

sms:+447911123456

A Chinese number is +86 followed by the 11-digit number:

sms:+8613812345678

Always include the country code. Without it, the message goes to a local number which might not be reachable internationally.

Multiple Recipients

The SMS URI can include multiple recipients separated by commas:

sms:+15551234567,+15559876543?body=Bulk%20message

This creates a group message with all recipients. Note: not all messaging apps support this, and group SMS behaves differently on iOS and Android (iOS creates an MMS group, Android creates a true SMS group).

SMS QR vs Native App

For collecting opt-ins and running text-based campaigns, SMS QR codes are often a better choice than building a native app. The reasons:

  • No app to download. The user already has a messaging app.
  • Works on every phone. iOS, Android, even basic flip phones.
  • Lower friction. One scan, one tap to send.
  • Easier to deploy. No app store approval, no version management.

Use SMS QR codes when the user action is simple (opt in, send a keyword, get info). Use a native app when the user action is complex (browse products, manage account, complete a multi-step process).

Compliance Considerations

SMS marketing is regulated in most countries. Before deploying SMS QR codes at scale, understand the rules:

US: TCPA (Telephone Consumer Protection Act)

You need express written consent before sending marketing text messages. The user opting in by scanning the QR code and sending the message can count as consent, but you should:

  • Have a clear privacy policy explaining what messages they\u2019ll receive
  • Include opt-out instructions in every message (\u201cReply STOP to unsubscribe\u201d)
  • Honor opt-outs immediately
  • Keep records of consent for at least 4 years

EU: GDPR

Similar requirements. You need explicit, informed consent. The opt-in must be clear about what data you collect, how you use it, and how to opt out.

Other countries

Check local regulations. Canada (CASL), Australia (Spam Act), and most other countries have similar rules.

This guide isn\u2019t legal advice. Consult a lawyer if you\u2019re running a large SMS campaign.

Frequently Asked Questions

What\u2019s the difference between sms: and smsto:?

The standard URI scheme is sms:. Some older iOS apps used smsto: but it\u2019s not standard. Always use sms: for compatibility.

Can I track scans of an SMS QR code?

Not directly through the QR code. Track via the messaging platform \u2014 count the messages received at the target number, or use a unique keyword per campaign to identify the source.

What if the user doesn\u2019t have a messaging app?

Rare. Every phone has at least a default messaging app. If your audience might be on tablets or other devices without SMS support, consider a web form fallback.

Can I include multiple keywords in the body?

Yes, just URL-encode the body properly. The user can edit before sending. Common pattern: pre-fill with the keyword, let the user add what they need.

Does SMS QR work on iPad?

iPads without cellular don\u2019t have a phone number, so SMS won\u2019t work. iPads with cellular do support SMS. If your audience is iPad-only, use a web form or email instead.

Conclusion: The Lowest-Friction Way to Get a Text

SMS QR codes work because they remove every step between the user and the action. No app to download, no number to type, no message to compose. One scan, one tap to send. The phone number and message are pre-filled, so the user is just one button press away from texting you.

Use them for promotions, support, opt-ins, and any context where you want a fast path to a real text message. Keep the body short, use international format for the phone number, and test on multiple phones. The 5-minute setup pays back the first time someone texts you instead of walking away.
