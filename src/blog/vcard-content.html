What Is a vCard QR Code?

A vCard QR code is a regular QR code that, when scanned, prompts the phone to save a contact. It works by encoding a small block of text in the vCard format, which is a universally accepted standard for digital business cards:

BEGIN:VCARD\nVERSION:3.0\nFN:Jane Cooper\nORG:Acme Industries\nTITLE:Head of Partnerships\nTEL;TYPE=WORK,VOICE:+14155551234\nEMAIL:jane@acme.com\nURL:https://acme.com\nEND:VCARD

When someone points their camera at this code, their phone reads it as a contact file and offers to add it. On iOS 11+ and Android 10+, this happens natively. No app, no signup, no friction.

Why it works on every phone

vCard has been around since 1995. It\u2019s the format your phone\u2019s Contacts app already speaks. Apple and Google both built QR-based vCard import into their default camera apps over the past few years, which is why this works without any third-party app. Over 95% of smartphones in active use today can scan a vCard QR code out of the box.

5 Best Practices for a vCard QR Code

Most \u201cmy contact info didn\u2019t save right\u201d complaints come from one of these five mistakes. Avoid them and you\u2019ll be fine.

1. Use the most current vCard version

vCard has two versions in common use: 3.0 (from 1998, supported everywhere) and 4.0 (from 2011, more compact but less compatible). For maximum compatibility across all phones, use vCard 3.0. If you\u2019re building for a controlled environment (like an enterprise app), vCard 4.0 saves bytes. For a public-facing QR code, stick with 3.0.

2. Use the TYPE parameter for phone numbers and emails

vCard lets you label phone numbers as CELL, WORK, HOME, FAX, etc., and emails as WORK, HOME. This helps phones sort them into the right place. For example:

TEL;TYPE=WORK,VOICE:+14155551234\nEMAIL;TYPE=WORK:jane@acme.com

Without these labels, the contact saves correctly but everything goes into a generic \u201cother\u201d field, which looks messy on the recipient\u2019s phone.

3. Keep error correction at H if you add a logo

QR codes have four error correction levels: L (7%), M (15%), Q (25%), H (30%). The higher the level, the more damage the code can sustain and still scan. If you put a logo in the center, you\u2019re literally covering part of the code. Set the error correction to H so the rest of the code can compensate.

4. Don\u2019t add a logo bigger than 30% of the code

The 30% rule is widely cited because QR codes have a lot of redundant data. Above 30%, you start losing too much information, especially at lower error correction levels. A small logo (15\u201325% of the code) is the sweet spot for visual impact and scannability.

5. Be careful with profile photos

vCard 3.0 supports an embedded photo via the PHOTO field. This sounds nice in theory \u2014 the contact saves with a face \u2014 but in practice, it bloats the QR code to a size that can be hard to scan. Most QR code generators either skip the photo or downscale it to a tiny thumbnail. For the best scan reliability, skip the photo in the QR code and let people put a face to the name on LinkedIn instead.

Frequently Asked Questions

Is a vCard QR code private?

The contact info you encode is public to anyone who scans the code. So treat the QR code like a digital business card \u2014 put it where you want people to scan it, not where random passersby can grab a photo. For a personal contact card, this is fine. For a more private contact method, consider a link-based QR code (like a Linktree or your own contact page) instead of an embedded vCard.

Do vCard QR codes work on iPhone?

Yes. iOS 11 and later (so iPhone 5S and newer, released 2013 onward) can read vCard QR codes with the default Camera app. No third-party app required.

Do they work on Android?

Yes. Android 10 and later (released 2019 onward) supports vCard QR codes in the default camera or Google Lens. Some older Android phones need a third-party scanner app, but those are increasingly rare.

vCard 3.0 vs 4.0: which should I use?

Use vCard 3.0. It\u2019s supported on every phone in active use. vCard 4.0 is more compact but has spotty support \u2014 some older Androids won\u2019t recognize it. The size difference for a typical business card is only a few dozen characters; it\u2019s not worth the compatibility risk.

How much info can a vCard QR code hold?

It depends on the QR code version. The largest standard QR code (Version 40 with error correction L) holds up to 2,953 bytes of data, or about 4,000 alphanumeric characters. In practice, a vCard with all standard fields fits comfortably in any error correction level. If you add a photo, the QR code gets significantly larger and harder to scan.

Do vCard QR codes expire?

No. A QR code is just encoded text. It doesn\u2019t point to a server, it doesn\u2019t have an expiration date. The only thing that changes is your contact info \u2014 and you can update the code anytime without changing the placement.

Can I include multiple phone numbers or emails?

Yes. vCard supports multiple entries for most fields. Just list them on separate lines with appropriate TYPE labels:

TEL;TYPE=CELL:+14155551111\nTEL;TYPE=WORK,VOICE:+14155551234\nEMAIL;TYPE=WORK:jane@acme.com\nEMAIL;TYPE=PERSONAL:jane@gmail.com

Phones handle this gracefully \u2014 multiple numbers and emails each get their own field in the contact.
